## 📝 다음에 해야 할 작업

### 1단계: 비밀번호 설정 (중요!)

`.env` 파일에서 로그인 비밀번호를 설정해야 합니다.

**방법**: `backend` 폴더의 `.env` 파일을 열어서 다음을 수정하세요:

```env
# 이 부분을 원하는 비밀번호로 변경하세요
SCHOOL_PASSWORD="원하는_학교_비밀번호"
ADMIN_PASSWORD="원하는_관리자_비밀번호"
```

**예시**:
```env
SCHOOL_PASSWORD="학교123!"
ADMIN_PASSWORD="관리자123!"
```

> **참고**: 이 비밀번호로 로그인합니다!
> - `SCHOOL_PASSWORD`: 일반 교직원 로그인 비밀번호
> - `ADMIN_PASSWORD`: 관리자 로그인 비밀번호 (교직원 관리, 연수 관리, 통계 메뉴 사용 가능)

---

### 2단계: 서버 실행

#### 백엔드 서버 실행 (터미널 1번)

```bash
cd backend
npm run dev
```

성공하면 다음과 같은 메시지가 나타납니다:
```
서버가 포트 3000에서 실행 중입니다.
```

#### 프론트엔드 실행 (새 터미널 창에서 - 터미널 2번)

```bash
cd frontend
npm run dev
```

성공하면 다음과 같은 메시지가 나타납니다:
```
  VITE v5.0.8  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

---

### 3단계: 브라우저에서 접속

1. **브라우저 열기**
   - Chrome, Safari, Firefox 등 아무 브라우저나 열기

2. **주소 입력**
   - 주소창에 입력: `http://localhost:5173`
   - 또는 `http://localhost:5173/login`

3. **로그인**
   - 비밀번호 입력 (`.env` 파일에서 설정한 비밀번호)
   - 일반 사용자: `SCHOOL_PASSWORD`로 설정한 비밀번호
   - 관리자: `ADMIN_PASSWORD`로 설정한 비밀번호

4. **시작하기**
   - 로그인 성공하면 대시보드가 나타납니다!
   - 관리자로 로그인하면 교직원 관리, 연수 관리, 통계 메뉴 사용 가능
   - 일반 사용자로 로그인하면 내 연수 메뉴만 사용 가능

## 🎯 사용 방법

### 관리자로 로그인한 경우

1. **교직원 관리**
   - "교직원 관리" 메뉴 클릭
   - "교직원 등록" 버튼 클릭
   - 이름, 이메일, 유형(교원, 직원, 공무직 등) 입력
   - 저장

2. **연수 관리**
   - "연수 관리" 메뉴 클릭
   - "연수 등록" 버튼 클릭
   - 연수명, 대상자 범위, 이수 기한 등 입력
   - 대상자 범위 선택 시 해당 유형의 교직원이 자동으로 연수에 포함됨
   - 저장

3. **연수 취합**
   - "연수 관리" 메뉴에서 연수 선택
   - "취합" 버튼 클릭
   - 각 교직원의 이수번호 입력
   - 상태 확인 (완료/미완료)

4. **통계**
   - "통계" 메뉴 클릭
   - 연수별 이수 현황 확인
   - 미이수자 목록 확인

### 일반 사용자로 로그인한 경우

1. **내 연수**
   - "내 연수" 메뉴 클릭
   - 본인이 참여한 연수 목록 확인
   - 이수번호 입력 (연수 완료 시)
   - 이수 완료 상태 확인

## 📧 이메일 설정 (선택사항)

리마인더 기능(연수 종료 3개월 전, 1개월 전 알림)을 사용하려면 이메일 SMTP 설정이 필요합니다.

### Gmail 사용 방법

1. **Gmail 앱 비밀번호 생성**
   - Google 계정 설정 접속: https://myaccount.google.com/security
   - "2단계 인증" 활성화 (아직 활성화하지 않은 경우)
   - "앱 비밀번호" 생성
   - 생성된 앱 비밀번호 복사

2. **`.env` 파일 수정**
   ```env
   SMTP_USER="your-email@gmail.com"
   SMTP_PASS="생성한_앱_비밀번호"
   ```

3. **서버 재시작**
   - 백엔드 서버를 중지하고 다시 시작
   - 이제 리마인더 이메일이 자동으로 발송됩니다

## 🔧 문제 해결

### 서버가 실행되지 않는 경우

1. **포트가 이미 사용 중인 경우**
   - 포트 3000이나 5173이 이미 사용 중일 수 있음
   - 다른 프로그램 종료 또는 포트 변경

2. **의존성 설치 오류**
   ```bash
   # 프론트엔드 재설치
   cd frontend
   rm -rf node_modules package-lock.json
   npm install

   # 백엔드 재설치
   cd ../backend
   rm -rf node_modules package-lock.json
   npm install
   ```

### 로그인이 안 되는 경우

1. **비밀번호 확인**
   - `.env` 파일의 `SCHOOL_PASSWORD`와 `ADMIN_PASSWORD` 확인
   - 비밀번호에 특수문자가 있는 경우 정확히 입력

2. **서버 재시작**
   - 백엔드 서버를 중지하고 다시 시작
   - `.env` 파일 변경 후 서버 재시작 필요

### 데이터베이스 연결 오류

1. **연결 문자열 확인**
   - `.env` 파일의 `DATABASE_URL` 확인
   - Neon 대시보드에서 연결 문자열이 올바른지 확인

2. **데이터베이스 재연결**
   ```bash
   cd backend
   npm run db:push
   ```
