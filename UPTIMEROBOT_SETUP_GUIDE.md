# UptimeRobot 설정 가이드

Render 무료 플랜에서 절전 모드로 인한 로그인 실패를 방지하기 위한 UptimeRobot 설정 방법입니다.

## 📋 UptimeRobot이란?

UptimeRobot은 **무료** 웹사이트/서버 모니터링 서비스입니다:
- 5분마다 서버에 요청을 보내서 서버가 살아있는지 확인
- 서버를 계속 깨어있게 유지 → 절전 모드 진입 방지
- 다운타임 발생 시 이메일로 알림 제공

## 🚀 설정 방법 (단계별 가이드)

### 1단계: UptimeRobot 계정 생성

1. **UptimeRobot 웹사이트 접속**
   - https://uptimerobot.com 접속
   - 우측 상단 "Sign Up" 클릭

2. **회원가입**
   - 이메일 주소 입력
   - 비밀번호 설정 (8자 이상)
   - "Create Account" 클릭
   - 이메일로 인증 메일 확인 (받은편지함 또는 스팸함 확인)

### 2단계: 모니터 추가

1. **대시보드 접속**
   - 로그인 후 대시보드 화면으로 이동

2. **새 모니터 추가**
   - 화면 상단 "+ Add New Monitor" 버튼 클릭
   - 또는 "Add Monitor" 버튼 클릭

3. **모니터 설정 입력**

   **Monitor Type (모니터 타입)**
   - ✅ **HTTP(s)** 선택 (기본값)

   **Friendly Name (모니터 이름)**
   - 입력 예시: `의무연수 플랫폼 서버`
   - 원하는 이름으로 입력 가능

   **URL (or IP)**
   - 입력: `https://studycheck.onrender.com/api/health`
   - ⚠️ 중요: 정확히 입력하세요 (https 포함, /api/health 경로 포함)

   **Monitoring Interval (모니터링 주기)**
   - ✅ **5 minutes** 선택 (무료 플랜의 최소 주기)
   - 5분마다 서버에 요청을 보내므로 절전 모드 진입 방지

   **Alert Contacts (알림 설정)**
   - 이메일 알림을 받고 싶으면 이메일 주소 추가
   - 또는 "Add Later" 선택

4. **저장**
   - 하단 "Create Monitor" 버튼 클릭

### 3단계: 확인

1. **모니터 목록 확인**
   - 대시보드에서 방금 추가한 모니터가 보임
   - 상태가 "Up" (초록색)으로 표시되면 정상

2. **테스트**
   - 약 5분 후 모니터 상태가 "Up"인지 확인
   - 서버가 계속 깨어있게 유지되는지 확인

## ✅ 설정 완료 후 효과

- ✅ 서버가 5분마다 요청을 받으므로 절전 모드로 진입하지 않음
- ✅ 로그인이 항상 빠르게 작동 (Cold Start 없음)
- ✅ 서버 다운타임 발생 시 이메일 알림 받을 수 있음

## 🔍 확인 방법

### 1. UptimeRobot 대시보드에서 확인
- 대시보드의 모니터 상태가 "Up" (초록색)인지 확인
- "Uptime"이 높은 수치(99% 이상)인지 확인

### 2. 직접 테스트
1. 15분 이상 서버에 접속하지 않음
2. 로그인 페이지로 이동
3. 로그인 시도
4. 빠르게 로그인되는지 확인 (5초 이내)

## 📝 참고사항

### UptimeRobot 무료 플랜 제한
- 최소 모니터링 주기: 5분
- 최대 모니터 수: 50개
- 월간 API 호출: 무제한 (충분함)
- 알림: 이메일, SMS, Slack 등 (제한적)

### 비용
- **완전 무료** (무료 플랜만으로도 충분)

### 대안 서비스 (참고)
- **Pingdom** (유료)
- **StatusCake** (제한적 무료)
- **Freshping** (무료, 50개 모니터)

## 🆘 문제 해결

### 모니터가 "Down"으로 표시되는 경우
1. URL이 정확한지 확인: `https://studycheck.onrender.com/api/health`
2. Render 서버가 실행 중인지 확인
3. CORS 설정 확인 (health 엔드포인트는 CORS 제한 없음)

### 여전히 절전 모드가 되는 경우
1. 모니터링 주기가 5분으로 설정되어 있는지 확인
2. 모니터가 "Up" 상태인지 확인
3. 5분 이내에 실제 사용자 요청이 있어야 함

## 📞 지원

문제가 발생하면:
1. UptimeRobot 공식 문서: https://uptimerobot.com/api/
2. Render 대시보드에서 서버 상태 확인
